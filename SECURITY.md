# Security Policy

TriMate's public repository is a capability package. It should contain coaching rules, skill definitions, and public reference notes only.

## Do Not Publish

Do not add:

- Real user training, body status, health, account, or conversation data.
- Device tokens, API keys, webhook secrets, model credentials, or private runtime configuration.
- Screenshots, logs, exports, backups, or local machine paths.
- Runtime architecture, deployment files, database schemas, or development test material.

## Reporting Issues

Please report security or privacy issues through GitHub security advisory instead of opening a public issue with private data.

If private data or credentials are published by mistake, rotate the credential, remove the data from git history, and treat it as exposed.
