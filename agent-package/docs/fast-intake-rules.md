# 快速记录识别规则

目标：用户发训练或身体状态时，智能体能快速识别、直接记录、少追问。

## 直接记录

出现以下任一内容，直接当作记录：

- 训练动作 + 距离：跑了 8 公里、骑车 40 公里、游泳 1500 米。
- 训练动作 + 时长：轻松跑 30 分钟、骑台 1 小时。
- 指标：配速、心率、功率、爬升、踏频、泳姿、划频。
- 身体状态：累、睡不好、压力大、酸痛、疼痛、恢复差。
- 目标：比赛日期、目标成绩、每周可训练时间。

## 默认推断

- 没说日期：今天。
- 没说项目但有“跑/配速”：跑步。
- `5k`、`5km`、`5 公里`：5 公里。
- 跑步语境里的 `530`、`5:30`、`5分30`：5:30/km。
- 游泳语境里的 `1500`：1500 米。
- “有点累/还行/很轻松/爆了”：主观感受，不要求数字化。
- “没疼/无痛/没有不适”：无疼痛。

## 才追问

每条记录最多追问 1 个问题。

只在这些场景追问：

- 完全不知道项目、距离和时长。
- 提到疼痛，但没有疼痛部位。
- 疼痛加重、改变动作、胸闷、头晕、呼吸异常。
- 用户要趋势分析，但没有说明时间范围。

## 回复模板

无需追问：

```text
已记录：今天跑步 8 公里，45 分钟，体感有点累，无疼痛。
建议：明天先轻松恢复，不加强度。
```

需要追问：

```text
已记录：今天跑步 8 公里，右腿疼。
为了判断风险，我只确认一个点：疼痛是在膝盖、小腿，还是脚踝？
```
