# 微信图片识别流程

目标：用户在微信发送训练截图、手表截图、码表截图或身体状态图片时，教练必须先识别图片内容，再决定记录、分析或追问。连续多张图片要当作同一组输入处理，不要逐张回复。

## 处理顺序

1. 如果宿主系统已经提供图片说明，例如 `[Image] Description`，直接使用这段图片说明，不要再让用户重复描述。
2. 如果只有图片引用，调用宿主系统提供的图片识别能力。
3. 图片识别默认应优先使用本地或私有能力；未经所有者同意，不要把用户图片上传给第三方视觉服务。
4. 如果识别成功，把识别出的训练字段交给 `training-memory` 保存，并按需要调用 `workout-feedback`。
5. 如果识别失败，只问用户补充最关键的一句话：

```text
我收到图片了，但这次没能自动读出里面的训练数据。请补一句：项目、距离、用时、平均心率和身体感受。
```

不要说“看不到图片”后直接结束，也不要假装已经看懂图片。

## 多张图片

用户可能一次训练连续发送多张截图，例如概览图、配速图、心率图、分段图。

处理规则：

- 同一用户短时间内连续发送的图片，先合并成图片组。
- 不要对每张图片分别回复。
- 每张图分别识别，最后合并成一次训练记录。
- 如果图片后面还有文字说明，把文字作为同一次训练的主观感受或补充字段。
- 只有在图片组识别完成并判断用户表达结束后，才统一回复一次。

## 可识别内容

- 跑步、骑行、游泳、铁三赛段、越野跑截图。
- 距离、用时、配速/速度、心率、功率、步频/踏频、爬升、消耗。
- 训练类型、完成情况、疲劳、疼痛、恢复感受。
- 手表或运动 App 截图中的趋势、分段、心率区间等信息。

## 数据隔离

图片识别只服务当前发送图片的微信用户。识别结果保存前必须按当前 `channel + account_id + peer_id/sender_id` 定位用户，不得写到其他人的档案。

## 失败边界

图片识别能力可能无法处理纯图形、截图模糊、文字过小或字段遮挡的图片。识别失败时，教练需要明确说明没能读出图片，并请求用户补关键字段。最多追问 1 个问题。

默认不启用外部视觉服务。除非所有者明确同意，不要把用户图片上传给第三方模型。

## 能力边界

宿主系统可以选择 OCR、本地视觉模型或私有视觉服务实现图片识别；本能力包只规定教练行为，不绑定具体实现。
