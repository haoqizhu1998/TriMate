# 数据、估算与信心

## 读取范围

先确认稳定用户身份。所有档案、训练、身体状态、目标和评估摘要都必须通过当前 `user_id` 读取和保存。

- 最近 6 周：逐周训练量、时长、强度、关键课、可比表现、疲劳、睡眠、压力、疼痛和恢复。
- 过去 3 个月：训练量、静息心率、能力和恢复的个人基线。
- 历史最佳：能力背景，不代表当前可用能力。
- 目标赛事：项目、日期、优先级和当前训练阶段。
- 上次评估摘要：只用于比较结论变化，不替代原始证据。

无法确认身份时，不读取任何长期记录，也不保存评估。

## 数据优先级

能力指标的可信顺序：

1. 正式比赛或标准测试。
2. 近期高质量、条件可比较的关键训练。
3. 设备估算。
4. 普通训练推算。

每个指标都注明：

- 数值和日期。
- 数据来源。
- 实测还是估算。
- 与上次可比数据相比的方向。

不要把轻松跑、恢复骑或技术游的偶然最快分段当成正式能力。

## 缺失数据

- 缺少一两项时继续评估。
- 可以合理推断时，明确写“这是估算值”及依据。
- 无法推断且不影响安全结论时留空。
- 只有会显著改变安全结论时才追问；每次最多追问一个问题。
- 不要求用户一次填完整表。

只有 1–2 周历史时：

- 可以判断当前恢复和明显红旗。
- 不轻易判断长期进步或平台。
- 行动建议偏保守。
- 信心固定为低，并列出需要继续积累的数据。

## 数据冲突

冲突时不要挑选更乐观的一条：

- 同时展示主要正面和负面证据。
- 优先检查数据日期、测试性质、环境和设备来源。
- 当前能否加量以近期可用状态和安全信号为准。
- 历史最佳只作为背景。
- 采用更保守的行动结论。

## 信心等级

### 高

- 至少 6 周连续训练和恢复数据。
- 有可信测试或比赛。
- 关键趋势之间基本一致。

### 中

- 有 3–6 周可用数据。
- 至少有一项可信能力依据。
- 存在少量缺失或环境不可比。

### 低

- 只有 1–2 周数据。
- 主要指标来自普通训练推算。
- 缺少年龄、性别、近期成绩或恢复基线。
- 多项证据互相冲突。

不要输出看似精确的信心百分比。

## 人群定位

使用 `入门 / 进阶 / 竞技`：

- 优先按年龄段、性别和专项成绩比较。
- “竞技”指同年龄、同性别业余人群中的高水平，不表示职业水平。
- 缺少人口或可信成绩数据时，结合现有成绩、训练年限和训练量给暂定定位，并标注低信心。
- 不凭单一训练量决定竞技水平。
- 体重只用于 FTP/W、跑步负担和异常体重变化，不评价身材。

## 保存字段

结构化评估摘要至少包括：

- `assessment_date`
- `window_start` 与 `window_end`
- `training_state`
- `action`
- `confidence`
- 指标值、趋势、来源和 `estimated`
- 各项目或分项结论
- 红旗代码和证据摘要
- 建议增量或恢复条件

不要默认保存完整自然语言报告。
