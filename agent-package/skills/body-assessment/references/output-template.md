# Markdown 输出模板

默认控制在 500–800 字，结论优先。没有数据的字段可以省略，不要用空表格撑满报告。

## 通用模板

```markdown
# 当前身体状态评估

> **结论：维持。** 训练状态仍在进步，但恢复信号开始变差；现在继续加量的收益可能小于疲劳风险。

- 训练状态：进步 / 平台 / 退步 / 疑似过载
- 行动建议：可加量 / 维持 / 该恢复
- 评估信心：高 / 中 / 低
- 评估有效性：反映今天的状态；出现新疼痛、疾病、关键比赛或测试后需重评

## 关键指标

- 跑步 VDOT：约 47（估算值，来自 2026-05-20 的 10K 训练）
- 骑行 FTP：245W，3.3 W/kg（实测，较上次稳定）
- 游泳 CSS：1:48/100m（标准测试，近期改善）
- 有氧基础：改善 / 稳定 / 下降
- 疲劳与恢复：正常 / 偏高 / 明显不足
- 近期负荷：稳定 / 增长较快 / 下降
```

## 医学硬红线模板

```markdown
# 当前身体状态评估

> **结论：停止相关训练，并及时寻求专业医疗评估。**

你描述的胸痛、接近晕厥、异常呼吸、明显心律异常，或疼痛加重并改变动作，属于不能靠训练调整继续观察的信号。

- 行动建议：该恢复，但这里不提供自行恢复期限
- 现在不要做：继续训练、强度课、长距离或低强度试跑
- 下一步：尽快接受合格专业人士评估

这不是医学诊断，但该信号需要优先排除健康风险。
```

## 表达规则

- 第一行必须让用户一眼看到 `可加量 / 维持 / 该恢复`。
- 红旗命中时直说，不用鼓励性语言稀释风险。
- 专业术语首次出现时用一句大白话解释。
- 不给精确成功概率。
- 不把免责声明当成风险控制。
