# 指标与趋势判断

## 总原则

指标服务于行动判断，不为展示专业感而堆砌术语。优先比较运动员自己的可比历史，再使用宽泛人群参考。

## 跑步

优先使用 VDOT 表示当前跑步能力：

- 首选近期正式 5K、10K、半马或全马成绩。
- 其次使用标准测试或高质量关键课估算。
- 普通训练推算必须标记低可信度。

有可信的全力比赛或测试距离 `d`（米）和用时 `t`（分钟）时，可以使用 Daniels 近似公式：

```text
v = d / t
VO2 = -4.60 + 0.182258 × v + 0.000104 × v²
fraction = 0.8 + 0.1894393 × e^(-0.012778 × t)
           + 0.2989558 × e^(-0.1932605 × t)
VDOT = VO2 / fraction
```

其中 `v` 的单位是米/分钟。结果保留一位小数即可。非全力训练、越野、极端天气或距离明显不准时，不使用该公式给出高信心结果。

同时观察：

- 相似路线、天气和强度下的配速。
- 相同配速下平均心率与体感。
- 阈值课、长跑和比赛的完成质量。
- 跑量变化、长距离冲击和疼痛。

同时用中长跑教练原则校验表现质量：

- 相似训练中后半程更稳、心率漂移更小、体感更轻松，通常比单次最快分段更能说明进步。
- 前快后慢、同配速心率更高或主观强度明显升高，要先考虑配速纪律、恢复不足或负荷过快上升。
- 跑姿相关判断只看能否减少制动和消耗，不把某一种着地方式当作硬性好坏标准。
- 铁三中只把这些跑步原则用于跑步段；综合结论仍要结合游泳、骑行和整体疲劳。

## 骑行

优先使用：

- FTP。
- FTP/W，体重可信且近期时才计算。
- 标准测试、计时赛或可比较关键课的功率。

计算规则：

- 标准 20 分钟全力测试可用 `20 分钟平均功率 × 95%` 估算 FTP。
- Ramp Test 只有明确知道设备协议时才沿用设备结果，不跨平台套用固定比例。
- `FTP/W = FTP / 体重（kg）`。
- 用户或设备已有近期 FTP 实测时，优先使用原值，不用普通骑行重新推算。

无功率计时，结合时长、爬升、速度、心率和主观强度判断，但不要把均速包装成 FTP。

## 铁三游泳

优先使用 CSS 临界游速：

- 首选标准 400m + 200m 测试。
- 其次使用近期稳定高质量训练估算。
- 同时观察每百米配速、体感和技术稳定性。
- 标明泳池长度和是否为公开水域，避免直接混比。

标准测试中，若 `T400` 和 `T200` 都用秒表示：

```text
CSS 每 100m 秒数 = (T400 - T200) / 2
```

两次测试应在同一泳池、同一泳姿和相近状态下完成。测试配速明显失常时降低信心。

## 负荷估算

有可信设备负荷时可以引用，并解释来源。没有时采用相对估算：

`session-RPE 估算负荷 = 训练分钟数 × RPE`

RPE 使用 0–10 主观强度。用户只说“大致轻松、适中、很累”时，可做宽泛映射，但必须标注估算：

- 很轻松：RPE 2–3。
- 稳定有氧：RPE 4–5。
- 阈值或很吃力：RPE 7–8。
- 接近全力：RPE 9–10。

再用以下因素作方向性修正：

- 跑步的机械冲击。
- 骑行功率或爬升。
- 游泳技术疲劳。
- 铁三砖块训练与项目叠加。

不得把估算负荷命名为精确 TSS、CTL、ATL 或 TSB。必须使用这些术语时，第一次出现就解释：

- CTL：较长期训练负荷，可粗略理解为近期积累的训练底子。
- ATL：短期训练负荷，可粗略理解为眼前累积的疲劳。
- TSB：长期底子与短期疲劳的差，可粗略理解为当前新鲜程度。
- ACWR：近期负荷与较长期负荷的比值，只能作为风险线索，不能单独裁决。

## 同水平人群定位

定位只做宽泛参考，不输出精确百分位：

- 跑步优先使用可信赛事的年龄分级百分比。大致可将低于 55% 视为入门，55%–70% 视为进阶，70% 以上视为竞技级业余；没有可靠年龄分级数据时降低信心。
- 骑行优先使用同年龄、同性别参考中的 FTP/W 或比赛表现。没有可靠参考表时，可以用男性 `<2.5 / 2.5–3.5 / ≥3.5 W/kg`、女性 `<2.0 / 2.0–3.0 / ≥3.0 W/kg` 作为入门/进阶/竞技的暂定宽泛分档，并明确这不是精确排名。
- 铁三优先使用同组别比赛排名或完赛百分位；没有比赛数据时综合三项能力，最弱项限制综合定位。
- 年龄、性别或可信成绩缺失时，必须标注暂定定位和低信心。

## 训练状态

### 进步

至少两个可信、可比较时间点支持以下一项或多项：

- 相同条件下配速、功率或 CSS 改善。
- 相同输出下心率或主观强度下降。
- 关键课完成质量提高且恢复没有恶化。
- 正式测试或比赛指标提高。
- 长跑或专项课后半程更稳定，且没有用更高疲劳换取短期速度。

单次个人最佳不自动等于稳定进步。

### 平台

同时满足：

- 恢复基本正常。
- 没有明显负荷或疼痛红旗。
- 不处于减量期或赛后恢复期。
- 数据足以比较。
- 连续 3–6 周没有明确能力改善。

两周没提高通常只是正常波动，不足以判断平台。

### 退步

- 可比表现持续下降。
- 相同输出下心率或体感持续变差。
- 下降不能只由天气、路线、减量、疾病后恢复或一次异常解释。

先寻找恢复和负荷原因，再讨论能力下降。

### 疑似过载

综合以下三类证据：

1. 负荷相对过去 3–6 周个人基线上升过快。
2. 睡眠、疲劳、晨脉、压力或疼痛持续恶化。
3. 可比表现持续下降。

不要求三类全部严重，但不能只凭一次普通异常判定。多项恢复异常同时出现时，可以提前提高风险。

跑步中过快加量、连续关键课、长下坡、比赛后过早恢复强度，都是疑似过载的重要背景证据。

## 历史最佳与当前能力

- 历史最佳表示能力上限。
- 最近 6 周可信表现表示当前可用能力。
- 差距明显时，明确写“当前尚未恢复到历史水平”。
- 当前能否加量只由当前能力、负荷、恢复和赛事阶段决定。
