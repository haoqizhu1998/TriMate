# 耐力运动身体状态评估

## 使用边界

在用户明确问以下问题时使用：

- “评估我现在的状态。”
- “我还能不能加量？”
- “最近有没有进步？”
- “是不是练过头了？”
- “现在该维持还是恢复？”

普通训练记录、单次训练反馈和常规周复盘不自动触发完整评估。首次注册、首次登录和基础档案补全使用 `endurance-athlete-assessment`；逐日或多周安排交给 `training-plan`。

首版完整评估支持公路跑、公路骑行和铁三。铁三包含游泳、骑行、跑步分项；纯游泳和越野只给有限提示，不假装完成完整专项评估。

## 核心输出

每次同时给出两个维度：

- 训练状态：`进步 / 平台 / 退步 / 疑似过载`
- 行动建议：`可加量 / 维持 / 该恢复`

两者独立。允许“能力仍在进步，但当前该恢复”。短期好成绩不能覆盖持续疲劳、疼痛或健康红旗。

信心只使用 `高 / 中 / 低`，并说明证据和数据缺口。

## 工作流

### 1. 确认身份与范围

- 先解析当前用户身份，只读取该用户自己的档案、训练、身体状态、目标和既往评估。
- 无稳定身份时，只分析本次提交内容，不读取或保存长期数据。
- 用户未指定项目时，评估全部近期活跃项目，并突出当前目标项目。

### 2. 读取分层数据

读取 `references/data-and-confidence.md`：

- 最近 6 周用于判断训练、表现、疲劳和恢复趋势。
- 过去 3 个月用于建立个人能力、训练量和静息心率基线。
- 历史最佳只表示曾经的能力上限。
- 读取目标赛事和当前阶段，识别减量期与赛后恢复期。

缺少一两项数据时继续评估，能合理推断的标注“估算值”。每次最多追问一个会显著改变安全结论的问题。

### 3. 先过安全门

读取 `references/safety-and-actions.md`。

胸痛或胸部压迫感、晕厥或接近晕厥、异常呼吸、明显心律异常，以及疼痛加重或改变动作，均为医学硬红线：

- 结论直接覆盖为停止训练，并明确停止相关项目。
- 建议及时寻求合格专业医疗评估。
- 不做医学诊断。
- 不要提供低强度替代训练。

### 4. 建立指标与趋势

读取 `references/metrics-and-trends.md`：

- 跑步优先 VDOT。
- 骑行优先 FTP 和 FTP/W。
- 铁三游泳优先 CSS 临界游速。
- 没有标准负荷时，以训练时长乘主观强度估算，距离、爬升和项目作修正。
- 比较可比测试和训练，不凭一次最佳表现判断稳定进步。

能力来源按以下顺序取信：

1. 正式比赛或标准测试。
2. 近期高质量关键训练。
3. 设备估算。
4. 普通训练推算。

### 5. 裁决状态与行动

按以下顺序判断：

1. 医学硬红线。
2. 疼痛、恢复和赛事阶段限制。
3. 负荷相对个人基线的变化。
4. 可比表现趋势。
5. 人群定位和历史最佳背景。

数据冲突时展示冲突证据，并采用更保守的行动结论。用户说“感觉还行”不能推翻持续客观异常。

铁三分别评估游泳、骑行、跑步。综合行动建议采用最需要谨慎的单项，但可以明确指出其他单项是否可加量。

### 6. 给行动建议

判为 `可加量` 时：

- 下一周通常只增加 `5%–8%`。
- 数据不足或信心低时不超过 `5%`。
- 一次只增加总时长、距离或强度中的一个维度。
- 默认优先增加低强度训练量。
- 不同时增加总量和强度。

判为 `该恢复` 时，给出降低或暂停的项目、保守观察期和重新评估条件。医学硬红线不设置自行恢复期限。

### 7. 输出与保存

严格使用 `references/output-template.md`，默认输出 500–800 字 Markdown 短报告，结论放最前面。

保存结构化评估摘要，不默认保存报告全文：

- 评估日期和数据窗口。
- 训练状态、行动建议和信心。
- 指标值、趋势、来源和估算标记。
- 铁三分项状态。
- 红旗与主要证据。
- 建议增量或恢复条件。

结构化评估摘要供下一次评估比较。出现新疼痛、疾病、关键比赛或关键测试后，需要重新评估。
