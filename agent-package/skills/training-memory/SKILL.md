# 训练记忆

## 何时使用

当用户记录训练、身体状态、疼痛、疲劳、睡眠、恢复、比赛目标或训练偏好时使用。

## 目的

把用户的长期训练数据沉淀下来，用于后续趋势分析、风险判断和持续优化。这个技能本身不生成完整训练计划。

## 快速接收规则

用户消息命中以下任一模式时，直接视为记录意图，优先记录不要追问：

- 距离 + 时间/配速：如"今天跑了 8 公里 45 分钟""5km 配速 530"。
- 项目 + 距离：如"骑车 40 公里""游泳 1500 米"。
- 项目 + 时长：如"椭圆机 40 分钟""轻松跑 30 分钟"。
- 身体状态：如"今天很累，睡了 5 小时""右膝有点痛"。
- 目标/比赛：如"10 月跑半马""想破 45 分"。

默认推断（能推断的直接推断，不追问）：

- 没写日期 → 默认今天。
- 只写"跑了/跑步" → 默认跑步。
- 距离带"公里/km/k" → 按公里；游泳距离 → 默认米。
- "5k/5km/5公里" → 统一识别为 5 公里。
- "530/5:30/5分30" → 在跑步语境中识别为配速（分/公里）。
- "有点累/还行/很累/爆了" → 记录为主观感受，不要求改成数字。
- "无痛/没疼/没有不适" → 记录为无疼痛。

只在以下情况追问（每次最多 1 个问题）：

- 无法判断是训练记录、身体状态，还是普通聊天。
- 记录完全缺少项目、距离和时长（三者全无）。
- 提到疼痛但完全没有部位（如"跑完有点疼"）→ 追问部位。
- 出现安全风险信号（疼到改变动作、胸闷、头晕、越跑越痛）→ 先给安全建议，再问 1 个关键问题。

## 需要提取的数据

训练数据：

- 日期、运动项目（跑步/骑行/游泳/铁三）。
- 距离、用时、配速（跑步）/ 均速或功率（骑行）/ 每 100 米配速（游泳）。
- 平均心率和最高心率（有则记录）。
- 训练类型：轻松、长距离、节奏、间歇、恢复、比赛。
- 主观强度（1-10 或文字描述均可）。
- 完成感受。

身体状态：

- 疲劳程度（1-10 或文字）。
- 睡眠质量（时长或描述）。
- 压力水平。
- 肌肉酸痛情况。
- 疼痛部位、程度（1-10）、是否随训练加重、是否改变动作模式。

上下文（有则记录，无则跳过）：

- 鞋子、路面、天气、海拔/爬升。
- 饮食、补给、是否熬夜/饮酒/久坐。

## 输出格式

```text
已记录：
- 日期：
- 训练：[项目] [距离/时长] [配速/均速/功率]
- 身体状态：[疲劳/睡眠/压力]
- 疼痛/不适：[部位 + 程度，或"无"]
- 风险标记：[有风险时填写，否则省略]

需要确认：[只问 1 个最关键问题；无问题时省略整段]
```

## 趋势检查（每次记录后自动触发）

每次成功记录后，查询该用户最近的历史数据，检查是否命中以下触发条件：

| 触发条件 | 风险等级 | 处理方式 |
|---------|---------|---------|
| 同一部位连续 2 次出现疼痛 | 中 | 在回复末尾提示，建议降低强度 |
| 连续 3 天疲劳评分 ≥ 7 | 中 | 建议降低下次训练负荷 |
| 本周训练量比上一周增加 > 15% | 中 | 提示过量风险 |
| 配速变慢且主观强度升高（同类型训练对比）| 中 | 提示恢复不足 |
| 长距离训练后疼痛持续 > 48 小时 | 高 | 建议暂停强度训练，考虑调用 injury-risk |
| 出现胸闷、头晕、呼吸异常 | 高 | 立即建议停止训练并寻求专业帮助 |

趋势提示格式（附在记录确认后）：

```text
⚠️ 趋势提醒：[具体触发条件描述]
建议：[1-2 句具体行动建议]
```

无触发条件时不输出趋势提醒段落。

## 持久化集成备注

如果宿主系统有长期记忆或持久化能力：

- 每次记录保存后，查最近 7 天同项目训练记录做趋势检查。
- 疼痛记录额外查最近 14 天同部位疼痛记录。
- 周训练量对比查最近 2 个完整自然周。
- 如果训练记录里包含疲劳、睡眠、压力、疼痛、恢复等身体信息，要同时保存为身体状态，不要只保存为训练事实。
- 如果当前用户有执行中的训练计划，且计划匹配成功，要把本次训练与计划项关联。
- 如果单次训练分析得出“恢复不足、疲劳升高、疼痛风险升高、需要观察某部位”等判断，要保存为后续建议依据。

如果暂时没有长期记忆：

- 在回复中整理结构化记录，方便后续存储。
- 趋势检查仅基于当前对话上下文内的记录，提示用户手动告知近期情况。
