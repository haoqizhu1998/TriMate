# 输出与计划预览模板

## 固定结构

第一次生成和重大改版必须按此顺序输出：

1. 计划状态与版本。
2. 目标赛事和优先级。
3. 当前能力、关键假设和置信度。
4. 目标可行性与风险。
5. 强度区间。
6. 周期阶段说明。
7. 完整周纲要。
8. 最近 4 周逐日安排。
9. 调整规则、预警和免责声明。
10. 一个确认问题。

## 顶部摘要

```markdown
# 训练计划预览 vX

- 状态：preview（确认后才生效）
- 周期：YYYY-MM-DD 至 YYYY-MM-DD
- 主赛：
- 热身赛：
- 每周可训练时间：
- 当前能力依据：
- 数据置信度：
- 本版关键假设：
```

## 风险双轨

目标合理时给单轨计划。目标偏激进或高风险但未触发医学硬红线时，必须展示：

```markdown
## 目标判断

- 风险等级：
- 为什么：
- 用户目标轨：按你的目标安排，但持续标出高风险周和退出条件。
- 安全建议轨：建议调整后的目标、周量或赛事优先级。
```

用户目标轨和安全建议轨都要可执行，不能只放一句免责文字。

## 区间

首次出现专业术语时解释：

```markdown
## 训练区间

- TSS：训练压力分数，用来统一比较不同训练的负荷。
- 跑步配速 / 骑行功率 / 心率区间：
- 大白话体感：
- 推算来源与置信度：
```

## 完整周纲要

从本周写到最后一个纳入规划的赛事。每周至少包含：

```markdown
### 第 N 周｜YYYY-MM-DD 至 YYYY-MM-DD｜基础期

- 本周目的：
- 目标 TSS：范围
- 大众指标：总时长；跑步公里/骑行时长/游泳距离（适用时）
- 关键课：
- 强度分布：轻松 / 阈值 / 高强度
- 恢复与力量：
- 对应赛事：
```

远期周次允许保持纲要，不要伪造过细的每日处方。

## 最近 4 周详细安排

最近 4 周逐日展开。少于 4 周则展开全部剩余时间。每次训练具有稳定训练 ID：

```markdown
#### 周二｜RUN-W02-D2-01｜阈值跑

- 目的：
- 热身：
- 主训练：
- 放松：
- 强度：配速/功率/心率范围 + RPE
- 预计时长/距离：
- 预计 TSS：
- 完成标准：
- 降级方案：
- 取消条件：
```

训练 ID 在移动日期或轻微调整时保持不变；训练目的发生根本变化时生成新 ID，并记录替换关系。

## 阶段解释

每个阶段用大白话解释，例如：“基础期不是练得慢，而是在把以后能承受关键课的底盘做厚。”

## 结尾

```markdown
## 执行与调整

- 每周根据完成度、疲劳、睡眠、疼痛和日程重新检查。
- 漏训不补债；必要时移动、降级或删除。
- 新增赛事或改变目标后生成新预览。

## 说明

本计划仅供参考，不替代医疗诊断或面对面的专业评估。请结合身体情况执行；出现胸部不适、晕厥、异常呼吸、明显心律异常，或疼痛加重并改变动作时，应停止训练并及时寻求合格专业人士帮助。

确认问题：你是否接受这版计划，并将它设为 active？
```

免责声明不能替代真实风险控制。
