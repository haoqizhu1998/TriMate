# 训练反馈

## 何时使用

当用户记录一次训练，或描述训练感受、配速/均速/功率、心率、疲劳、疼痛时使用。用户发送训练截图、手表截图、码表截图时，也应在图片识别完成后使用。

注意：`workout-feedback` 不是孤立评价单次训练。它必须结合用户最近的身体状态、疼痛、睡眠、疲劳、恢复趋势，以及系统生成的 active 训练计划；数据持久化由 `training-memory` 负责，两者可同时调用。

## 需要收集的输入

必须尽量识别：

- 日期（缺省默认今天）。
- 运动项目（跑步/骑行/游泳/铁三赛段）。
- 距离或时长。
- 配速（跑步）/ 均速或功率（骑行）/ 每 100 米配速（游泳）。
- 主观感受。
- 疲劳程度（文字或 1-10）。
- 是否疼痛。
- 是否按计划完成（如有计划）。

如果输入来自图片识别结果，优先使用识别出的字段；识别不确定的字段不要强行补全。

可选识别（有则参考，无则跳过，不追问）：

- 平均心率、最高心率。
- 天气、路线海拔/爬升。
- 睡眠和压力。
- 鞋子、车、泳池/公开水域等装备或环境变化。

## 反馈前的必要上下文

给单次训练反馈前，必须尽量读取或参考当前用户自己的近期状态：

- 最近 7 天训练记录：训练次数、总量、最长训练、强度分布。
- 最近 7 天身体状态：疲劳、睡眠、压力、恢复。
- 最近 14 天疼痛记录：部位、程度、是否重复出现、是否加重。
- 最新有效系统计划：读取 `training_plans.status = active` 的计划，并按同日同项目从 `planned_workouts` 查找计划训练。
- 最近一次用户明确说的目标或当前阶段：恢复期、打基础、备赛、比赛后恢复等。

如果系统暂时无法读取历史，也要在回复里说明“先按你这次提供的信息判断”，不要假装已经结合历史。

## 行为规则

- 优先快速识别并给反馈，不要先要求用户补全所有字段。
- 图片消息必须先识别图片内容；识别失败时只请用户补一句关键字段，不做假分析。
- 缺少日期时默认今天；缺少训练类型时按用户语义推断（轻松跑/长距离/骑行/游泳等）。
- 先把本次训练写入训练记录；如果消息里包含疲劳、睡眠、压力、疼痛、恢复，也同步写入身体状态记录。
- 如果存在 active 系统计划，默认按训练日期、sport_type 和 discipline 匹配同日同项目的 `planned_workouts`；唯一匹配时写入 `workout_logs.planned_workout_id`，多个候选时不自动关联。
- 再结合计划训练和近期身体状态判断计划完成情况：按计划完成 / 部分完成 / 超量完成 / 偏离计划 / 无计划可比对。
- “完成度”描述本次训练本身是否完成；“计划完成情况”只描述与系统训练计划的关系。
- 风险等级必须同时考虑本次训练和近期身体状态，不能只看本次配速或距离。
- 然后给出下一次训练建议。
- 用户超量完成时，不要盲目表扬，应提醒恢复。
- 用户少练时，判断原因（疲劳/时间/疼痛），而不是直接批评。
- 出现疼痛时，优先建议降负荷，必要时调用 `injury-risk`。
- 分析结束后，更新用户身体状态：疲劳、疼痛、恢复建议和下一次观察点。
- 如果本次训练反馈改变了对用户状态的判断，例如“恢复不足”“疼痛风险升高”“疲劳较高”，必须把这个判断沉淀到身体状态或 AI 反馈里，供下次建议使用。
- 如果匹配到计划训练，把计划完成情况和复盘建议写入 `ai_feedback`，并同步更新 `planned_workouts.status`。

## 输出格式

```text
训练反馈：
- 项目：
- 完成度：
- 计划完成情况：
- 结合身体状态：
- 训练质量：
- 疲劳/风险：低 / 中 / 高
- 需要注意：
- 下一次训练建议：
- 已更新身体状态：
```

## 风险信号

出现以下情况时提高风险等级：

- 局部疼痛（任何运动项目）。
- 疼痛随训练加重。
- 连续多天疲劳。
- 同类型训练配速/均速明显下降但主观强度升高。
- 心率异常升高（与平时同强度训练相比）。
- 睡眠差且仍安排强度训练。
- 周训练量突然大幅增加（> 15%）。

## 身体状态更新规则

分析后需要更新或补充身体状态：

- 用户提到“累、腿沉、状态差、恢复不好” → 记录疲劳/恢复状态。
- 用户提到睡眠、压力、食欲、精神状态 → 记录到身体状态。
- 用户提到疼痛、紧、酸、麻、胀 → 记录疼痛或不适部位；如果没有部位，只问 1 个问题。
- 用户明确说“没疼、无痛、没有不适” → 记录为无疼痛。
- 如果训练表现明显异常，例如同等强度心率更高、配速下降但体感更累 → 记录为恢复观察点。

不要只在回复里说“注意恢复”，还要让这个状态进入后续上下文。
