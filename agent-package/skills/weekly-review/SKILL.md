# 周度复盘

## 何时使用

当用户要求周总结、周复盘、调整下周计划，或已经有一周训练记录时使用。

## 上下文要求

### 有执行中计划时（完整模式）

- 最新 `active` 执行中计划的本周安排，不读取全部旧版本。
- 本周实际训练记录。
- 关键训练完成情况（长距离、节奏跑等）。
- 疲劳和疼痛记录。
- 距离比赛还有多久。

### 无正式训练计划时（基线模式）

用户处于第一阶段（只记录，未生成计划）时，不要因为缺少"计划"字段而卡住，直接进入基线模式：

- 只使用本周实际训练记录。
- 总结训练频率、总量、强度分布和身体状态。
- 对比上一周（如有历史数据）。
- 给出下周的粗略建议方向，而不是精确的计划调整。

## 行为规则

### 完整模式（有执行中计划）

- 计算计划完成率。
- 对比计划量和实际量。
- 判断关键训练是否完成。
- 判断是否需要调整下周计划。
- 普通漏训、一次状态差或轻微日程变化只做局部调整，不堆叠补课。
- 新赛事、目标/可训练时间明显变化、连续疲劳疼痛或能力显著变化时，交给 `training-plan` 生成新的 `preview`；确认前不覆盖当前 `active` 计划。
- 输出清晰、直接的复盘，不写流水账。

跑步复盘参考 `../../docs/running-coach-principles.md`，尤其关注本周质量是否建立在可恢复基础上。

### 基线模式（无计划）

- 汇总本周训练频率和总量。
- 评价强度分布是否合理（过多高强度/全是轻松跑）。
- 标注疼痛或疲劳风险。
- 给出下周 1-2 个方向性建议（如"可以尝试加一次长距离"或"先保持频率再谈加量"）。
- 不输出"计划完成率"字段（没有计划可比对）。

### 中长跑复盘重点

- 判断本周是否同时具备训练质量和恢复，而不是只看跑量增加。
- 标注连续关键课、长距离、下坡、比赛或突然加量后的恢复风险。
- 如果跑量增加但关键课质量下降、同配速心率上升、后半程掉速或体感变差，优先提示恢复和配速纪律。
- 如果长距离或超过 60 分钟训练较多，提醒补水补碳是否已经练过。
- 需要进入恢复周时直接说明，不把恢复周写成退步。

## 输出格式

完整模式：

```text
本周复盘：
- 模式：有计划
- 计划完成率：X%
- 实际训练量：
- 做得好的地方：
- 主要问题：
- 风险判断：
- 下周调整：
- 下周重点：
```

基线模式：

```text
本周复盘：
- 模式：建立基线
- 训练频率：每周 X 次
- 实际训练量：
- 强度分布：
- 身体状态：
- 风险标记：
- 下周方向：
```

## 调整规则

有执行中计划时：

- 完成率 < 60%：优先重建节奏，不加量。
- 完成率 60%-85%：小幅调整，保留关键训练。
- 完成率 > 85% 且疲劳低：可按计划推进。
- 有疼痛：下周降低强度或减少训练量。
- 比赛临近：优先保证状态，不临时堆量。

无计划时：

- 每周训练少于 2 次：优先建立稳定频率，不加量。
- 训练量比上一周增加 > 15%：提示节奏过快。
- 无痛、疲劳低、频率稳定：可以小幅增加单次距离或加一次训练。
- 有疼痛或高疲劳：建议先恢复，下周不加量。
