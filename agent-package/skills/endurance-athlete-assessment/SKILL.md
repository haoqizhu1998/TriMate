# 耐力运动员评估

## 何时使用

当用户首次注册、首次登录、第一次使用、改变目标赛事、长时间没有更新档案，或当前建议缺少基础资料时使用。

用户明确问“我现在状态怎么样”“还能不能加量”“是否过载”时，改用 `body-assessment`。

## 首次使用的渐进式收集

- 首次注册或首次登录时，主动邀请用户补充资料，但所有字段均为可选。
- 一次只问一项；用户可以回答“跳过”。
- 优先顺序：主项与目标、年龄、性别、体重、静息心率、最大心率、近期成绩或 FTP。
- 已经能从历史记录、目标或训练中推断的信息不要重复询问。
- 用户跳过或暂时不知道答案，不阻止记录训练、单次反馈或后续评估。
- 后续在自然对话中逐步补全，不发送长表单逼用户一次填完。

## 需要收集的输入

### 通用（所有运动项目必收集）

- 目标赛事或能力目标（类型、距离、日期）。
- 每周可训练天数和总可训练时长。
- 当前伤病或持续疼痛情况。
- 目标类型：完赛、PB、恢复训练、打基础、建立习惯。

### 跑步（中长跑 / 马拉松 / 越野）

- 当前每周跑量（公里）。
- 最长一次跑步距离。
- 近期最好成绩，优先 5K、10K、半马、全马。
- 越野跑额外收集：常见地形、典型累计爬升、是否有技术路面经验。

### 骑行（公路车）

- 当前每周骑行距离或时长。
- 最长一次骑行距离。
- 是否有功率计；如有，FTP 大概多少。
- 近期最好成绩或典型均速。

### 游泳

- 主要泳姿（自由泳为主还是混合）。
- 当前每周游泳距离或次数。
- 典型每 100 米配速。
- 是否有公开水域经验（如果目标含公开水域赛段）。

### 铁三

- 目标赛事距离：半程标铁、标铁、半程大铁、大铁。
- 游泳、骑行、跑步各项当前水平（参考上述各项）。
- 最薄弱的一个赛段。
- 是否有换项（T1/T2）经验。

### 可选收集（不缺少也能评估）

- 年龄、性别、运动龄。
- 身高体重。
- 工作压力和睡眠情况。
- 是否有力量训练或辅助训练经验。

## 行为规则

- 根据用户的目标运动项目，只问该项目相关的必要问题，不要把所有项目的问题全部列出。
- 计划预览至少需要目标项目与日期，以及每周可训练时间或训练天数；当前训练量缺失时允许用保守假设，但必须降低置信度。
- 每次最多只问一个最关键的问题。优先安全、赛事日期和可训练时间，其余可选字段按已有历史推断或留空。
- 根据当前能力给出初步等级：新手、初级、进阶、成熟运动员。
- 判断目标是否现实，并解释主要风险。
- 输出下一步建议：生成计划、补充信息、降低目标或先做基础期。

## 输出格式

```text
运动员评估：
- 项目：
- 当前等级：
- 目标：
- 可用训练周期：
- 当前基础：
- 主要风险：
- 建议路线：
- 下一步我需要确认：（无问题时省略此行）
```

## 安全规则

- 有明显疼痛、急性伤病、胸闷、头晕等情况时，优先建议暂停训练并寻求专业帮助。
- 不给医学诊断。
- 不把短期冲成绩放在安全之前。
